## TEST SUITE 1: HEADER FORMAT VALIDATION

### Research Question

**Can headers be generic natural language, or do they require formalization?**

### Hypothesis

Headers function on a spectrum from highly informal to highly formalized. We need to identify the minimum viable formalization level for reliable domain activation.

### Test Design

**Method:** Present identical cipher with progressively formalized headers to multiple systems and measure:
- Decode accuracy
- Interpretation consistency
- Domain activation reliability

---

## Header Formalization Levels

### Level 1: Casual/Conversational (Minimal)
**Examples:**
- "Think about music"
- "You're an engineer"
- "Consciousness stuff"

**Characteristics:**
- Natural conversational language
- Implicit domain reference
- Minimal technical framing

---

### Level 2: Descriptive (Low Formalization)
**Examples:**
- "Consider this from a musical perspective"
- "Interpret through cloud infrastructure concepts"
- "View this as consciousness theory"

**Characteristics:**
- Clear domain indication
- Still natural language
- Action-oriented framing

---

### Level 3: Declarative (Medium Formalization)
**Examples:**
- "Consciousness is fundamental"
- "Cloud infrastructure engineering"
- "Formal musical theory and composition"

**Characteristics:**
- Ontological or professional statement
- Clear conceptual framework
- Tested format (our current standard)

---

### Level 4: Parametric (High Formalization)
**Examples:**
- "DOMAIN=consciousness; FRAMEWORK=idealism; LENS=phenomenology"
- "DOMAIN=infrastructure; FRAMEWORK=distributed_systems; LENS=operations"
- "DOMAIN=music; FRAMEWORK=classical_theory; LENS=harmonic_analysis"

**Characteristics:**
- Structured key-value format
- Explicit parameter specification
- Programming language style

---

### Level 5: Formal Grammar (Maximum Formalization)
**Examples:**
- `<HEADER domain="consciousness" ontology="fundamental" method="introspective"/>`
- `<HEADER domain="infrastructure" ontology="technical" method="analytical"/>`
- `<HEADER domain="music" ontology="aesthetic" method="compositional"/>`

**Characteristics:**
- XML/markup format
- Fully structured metadata
- Machine-parseable
